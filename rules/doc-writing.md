# 문서 작성 규칙 — prometheus-practice

## 기본 원칙
1. 모든 문서는 한국어로 작성하되, 기술 용어(PromQL, Alertmanager, scrape 등)는 영어 그대로 사용
2. 제목은 `# [주제] — Prometheus [문서유형]` 형식으로 통일
3. 문서 상단에 작성일, 작성자, 검토자 메타데이터 포함
4. 코드 블록에는 반드시 언어 지정 (```promql, ```yaml, ```bash)
5. 모든 문서에 트러블슈팅 섹션 포함

## 문서 구조
- **개요**: 목적과 배경 (3-5문장)
- **구성**: 설정 정보 및 아키텍처
- **운영**: 핵심 명령어 및 체크리스트
- **트러블슈팅**: 증상-원인-해결 테이블
- **참고**: 관련 문서 링크

## PromQL 작성 규칙
- 쿼리에 주석으로 목적 설명
- 임계값은 주석으로 근거 명시
- recording rule 이름은 `level:metric:operation` 형식
- 알림 이름은 PascalCase (예: `PrometheusTargetDown`)

## 금지 사항
- 개인 식별 정보 포함 금지
- 프로덕션 자격증명 문서에 포함 금지
- 미검증 PromQL 쿼리 문서화 금지
